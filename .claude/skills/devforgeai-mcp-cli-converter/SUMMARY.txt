THE HYBRID APPROACH: 80/20 SPLIT
================================================================================

80% AUTOMATIC (Framework handles)
├─ Pattern 1: API Wrapper (stateless)     → stateless-api-adapter.py
├─ Pattern 2: State-Based (stateful)      → state-based-adapter.py
└─ Auto-generation of CLI and skill docs

20% CUSTOMIZABLE (You implement)
└─ Pattern 3: Custom (advanced/hybrid)    → Your custom adapter code
   For when patterns don't fit your needs

RESULT: Works for virtually any MCP

================================================================================
HOW IT WORKS
================================================================================

INPUT: MCP Server (from npm, local path, or schema file)
  ↓
MCPAnalyzer
  ├─ Extracts tool definitions
  ├─ Detects pattern (API wrapper, state-based, custom)
  ├─ Calculates confidence (0-1)
  └─ Provides recommendations

CLIGenerator (Pattern-Specific)
  ├─ API Wrapper Pattern: Direct 1:1 command mapping
  ├─ State-Based Pattern: Session lifecycle management
  └─ Custom Pattern: Your orchestration logic

SkillGenerator
  ├─ Creates SKILL.md (Claude knows how to use it)
  ├─ Creates references (CLI docs, examples)
  └─ Creates error codes & recovery steps

OUTPUT:
  ├─ ./cli.py (standalone CLI, runs immediately)
  ├─ ./skill/SKILL.md (skill Claude understands)
  └─ Both ready for DevForgeAI integration

QUICK START
================================================================================

1. ANALYZE AN MCP
   python scripts/converter.py analyze npm:mcp-puppeteer@latest
   → Detects pattern, confidence, recommendations

2. CONVERT TO CLI + SKILL
   python scripts/converter.py convert puppeteer \
     --source npm:mcp-puppeteer@latest \
     --output-dir ./puppeteer-cli
   → Generates CLI and skill

3. USE THE CLI
   python puppeteer-cli/cli.py session create
   python puppeteer-cli/cli.py navigate --session abc123 --url "https://..."

4. USE IN DEVFORGE
   /convert-mcp-server puppeteer-mcp
   → Dev persona instantly has puppeteer-cli available

================================================================================
THE THREE PATTERNS
================================================================================

PATTERN 1: API WRAPPER (Stateless)
├─ Use for: REST APIs, weather services, translation, payment processors
├─ Detection: Stateless tools, no session/connection keywords
├─ CLI Style: Direct command mapping (no state)
│  Example: weather-cli forecast --location Seattle --days 5
└─ 70% of MCPs fit here

PATTERN 2: STATE-BASED (Stateful)
├─ Use for: Browser automation, databases, file operations
├─ Detection: Session/connection/state keywords present
├─ CLI Style: Session lifecycle (create → use → destroy)
│  Example: puppeteer-cli session create
│          puppeteer-cli navigate --session $ID --url "..."
│          puppeteer-cli session destroy --session $ID
└─ 20% of MCPs fit here

PATTERN 3: CUSTOM (Hybrid/Advanced)
├─ Use for: Complex orchestration, caching, streaming, proprietary workflows
├─ Detection: Doesn't fit patterns 1 or 2
├─ Implementation: You provide custom adapter code
│  class CustomAdapter:
│    def execute(self, command, args): ...
└─ 10% of MCPs fit here (but framework makes it easy)

================================================================================
CONFIDENCE SCORING
================================================================================

The analyzer reports confidence (0-1) in pattern detection:

0.95+ → Very confident, trust it
0.85-0.94 → Confident, safe to use
0.70-0.84 → Moderate, consider reviewing
0.50-0.69 → Uncertain, may want to override
< 0.50 → Low confidence, use --pattern custom

Can always override: --force-pattern state-based

================================================================================
ERROR HANDLING
================================================================================

Generated CLIs use standard Unix exit codes:

0 → Success
1 → General error
2 → Invalid arguments / Usage error
3 → Timeout
4 → Resource unavailable / Connection error
5 → Authentication failed / Permission error
6 → Rate limited / Service unavailable

Skill automatically documents these and provides recovery steps.
